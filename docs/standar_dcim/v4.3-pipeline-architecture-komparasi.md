## 5. Rekomendasi prioritas selanjutnya

1. **Perbaiki dokumen** (cepat, tanpa sentuh sistem): update badan v4.3 ke kondisi nyata; koreksi klaim Kafka ACL di task.md (sama kasusnya dengan Flink).
2. **Migrasi ingestion Telegraf→NiFi** (keputusan user) — paralel-run bertahap; lihat prompt khusus.
3. **Struktur & config topik**: partisi 1→12, set `min.insync.replicas=2`, evaluasi tahap `validated`.
4. **Keamanan**: aktifkan Authorizer + Kafka ACL (RBAC nyata).
5. **Observability**: keputusan Prometheus/Grafana vs tetap Kibana (buat ADR).
6. **Data Quality Framework** + **real-time/Flink** (Opsi A) sebagai task terjadwal.

> Catatan: P1 keamanan/HA dari komparasi 2026-06-25 sudah selesai. Pola yang berulang dari agent sebelumnya: **menandai "selesai" tanpa bukti** (Flink, Kafka ACL). Untuk pekerjaan berikutnya, setiap "done" wajib disertai bukti verifikasi yang bisa direproduksi.
